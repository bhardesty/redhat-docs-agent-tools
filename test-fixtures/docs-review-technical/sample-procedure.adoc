= Installing the operator

[role="_abstract"]
Install the Example Operator on your OpenShift cluster using the CLI or the web console.

.Prerequisites

* You have access to an {product-title} cluster with `cluster-admin` privileges.
* You have installed the `oc` CLI tool version 4.14 or later.
* You have installed `jq` for JSON processing.

.Procedure

. Log in to your OpenShift cluster:
+
[source,terminal]
----
$ oc login https://api.example.com:6443 -u kubeadmin
----

. Create the namespace for the operator:
+
[source,terminal]
----
$ oc create namespace example-operator
----

. Apply the operator subscription:
+
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: example-operator
  namespace: openshift-operators
spec:
  channel: stable
  name: example-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
  config:
    tolerations:
      - key: node-role.kubernetes.io/infra
        effect: NoSchedule
----

. Verify the operator is running:
+
[source,terminal]
----
$ oc get pods -n example-operator --field-selector=status.phase=Running
----

. Check the operator version:
+
[source,terminal]
----
$ oc get csv -n example-operator -o jsonpath='{.items[0].spec.version}'
----

.Verification

* The operator pod is in `Running` state.
* The ClusterServiceVersion shows `Succeeded` phase.
